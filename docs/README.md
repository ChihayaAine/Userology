## 📝 文档规范

### 文档结构

每个功能模块文档应包含：

1. **概述** - 功能简介和目标
2. **技术实现** - 核心技术方案
3. **API 接口** - 相关 API 说明
4. **数据库** - 相关表结构
5. **前端组件** - UI 组件说明
6. **测试** - 测试方案和用例

### Markdown 规范

- 使用清晰的标题层级（# ## ### ####）
- 代码块标注语言类型
- 使用表格展示结构化数据
- 添加适当的 emoji 提升可读性
